Overall Architecture

Exo is a local AI workstation over a Markdown-first exograph.

The desktop app, CLI, and MCP bridge all talk to one resident local runtime. User knowledge stays in Markdown. Derived state, plugin state, transcripts, activity records, and artifacts live under `.exo`.

Surfaces

Desktop, CLI, and MCP are separate interfaces over the same local command server. CLI stays broad; MCP stays narrow.

Runtime

The app can run while the window is hidden. Resident services own command routing, terminals, watchers, plugin state, and settings.

Storage

Markdown remains canonical. `.exo` stores derived and operational state that should not pollute user-authored files.

Extensibility

Plugins declare capabilities and compose through core hosts rather than bypassing renderer, terminal, or web-viewer boundaries.

Daily-use invariant

  • Open and edit Markdown.
  • Search local context.
  • Run and supervise terminals/agents.
  • Preview local artifacts and web surfaces.

Architecture pressure

  • Keep terminal and web viewer as core safety boundaries.
  • Move harnesses, search providers, routines, evals, and profiles into plugins.
  • Expose enough command APIs for Exo-on-Exo without turning MCP into admin mode.
