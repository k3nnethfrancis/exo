Plugin Boundary

Core is the workstation substrate. Plugins are replaceable capabilities.

Vanilla Exo should be core plus bundled/recommended plugins. Disabling optional plugins must still leave a useful Markdown graph editor with basic search, terminal, web viewer, settings, and command surfaces.

Core substrate Bundled plugin External plugin Trust and permissions

Core

Stable, security-reviewed services every capability composes through.

Markdown graphFiles, links, tags, frontmatter, roots, basic search.
Pane and layout hostGrid, tabs, tool dock, editor bodies, persisted layout.
Terminal runtimetmux durability, xterm rendering, scrollback, transcripts, diagnostics.
Web viewer hostOpen local paths, localhost URLs, artifacts, trusted URLs.
Command planeDesktop command server, CLI contracts, MCP base contracts.
Registry and trustCapability metadata, permission grants, plugin settings surfaces.

Bundled Plugins

First-party capabilities that ship with Exo but use plugin-shaped seams.

QMD search providerAdvanced local retrieval behind a provider contract.
Harness adaptersShell, Claude Code, Codex, Pi. Hermes hidden until stable.
Routine templatesPrompt plus harness plus scope plus output policy.
Graph/profile helpersStarter profiles, analyzers, health reports, OKF-aware templates.
Tool dock surfacesLaunchers, config panels, provider controls, status widgets.

External Plugins

Optional packages that add capability variation without owning core surfaces.

Search alternativesGraph search, vector search, rerankers, remote retrieval.
Inference enginesllama.cpp, Ollama, vLLM, local APIs used by harnesses like Pi.
Dashboards and artifactsLocal web apps opened through the core web viewer.
Evals and tracesCollectors, exporters, scorers, replay views, datasets.
Profile packsOKF, Shoshin, LM Wiki, domain/project-specific graph rules.
