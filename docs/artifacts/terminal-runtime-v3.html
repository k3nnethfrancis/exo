Terminal Runtime V3

tmux owns durable process state. xterm owns the live screen.

The simplifying move is not to remove tmux. It is to stop duplicating terminal ownership across transcripts, React state, tail buffers, and hydration resets.

Core invariant

  • Only xterm interprets live terminal output.
  • Only tmux owns durable process and session state.
  • React state stores metadata, never the live screen.

Known failure class

  • Blank terminals after reload are hydration bugs.
  • Replacement glyphs point to control-mode decoding or font/charset paths.
  • Scrollback bugs usually mean tail, transcript, and xterm ownership got mixed.

Test gate

  • Fake Claude/Codex agents, not live inference.
  • Reload, relaunch, pane move, resize, Unicode, and large output.
  • Assert no mounted terminal calls read-and-reset during normal focus.
Read this architecture as a decision

Exo keeps tmux persistence because long-running agents and builds must survive normal workspace lifecycle. The refactor work is about reducing live-output ownership, not adding another terminal backend.
